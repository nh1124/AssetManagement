# 次世代金融資産管理システム定義書 (Asset Management System Definition 2.3)

## 1. 第1層: Governance (オーナーシップ / Why)
**定義:** システムや技術が変化しても変わらない、貴方の人生における資産管理の根本哲学と究極の目的。

*   **Vision (到達すべき状態):**
    *   **「精神的無限富裕 (State of Financial Infinity)」の確立。**
        *   物理的な無限ではなく、「お金がないことによる制約」や「将来への不安」が認知資源を占有しない状態。
        *   欲しいものや必要な体験に対して、資金不足を理由に躊躇することがない精神的自由。

*   **Mission (果たすべき任務):**
    *   **「生活防衛」と「人生品質向上」の同時達成。**
        *   予期せぬリスクに対する鉄壁の防御（Solvency）と、人生を豊かにするための積極的な資産拡大（Growth）を両立させる基盤を維持する。

*   **Core Values (行動指針):**
    1.  **Role-Driven Assets (全ての円に役割を):** 目的のない「死に金（アイドルマネー）」を許容しない。全ての資産は「防衛」か「増殖」、あるいは「特定の将来支出」のいずれかの任務を帯びる。
    2.  **Logical Balance over Actual Balance (実残高より論理残高):** 銀行口座の表面上の残高（Actual Balance）ではなく、クレカ引き落とし予定額や近々の固定費を差し引いた「真に使えるお金（Logical Balance）」を管理の正（Source of Truth）とする。
    3.  **Decision First (意思決定至上主義):** 「記録」は手段に過ぎない。全ての機能は、未来の行動（投資、消費、待機）を決定するために存在する。
    4.  **Long-Term Smoothing (長期的平準化):** 月次の収支変動に一喜一憂せず、数十年単位のライフサイクルコストで物事を捉える。

## 2. 第2層: Management (ストラテジー / What)
**定義:** ガバナンス層のビジョンを実現するための、CFO（最高財務責任者）的視点での具体的な戦略と管理ルール。

*   **Strategic Objectives (戦略目標):**
    1.  **流動性リスクゼロ化:** 突発的な支出（Unforeseen Liability）および既定の将来支出に対し、100%の支払い能力を常時確保する。
    2.  **TCO (Total Cost of Ownership) 最適化:**
        *   **Asset Management:** 物品購入を「浪費」ではなく「減価償却資産の取得」と定義し、耐用年数あたりのコストパフォーマンスを最大化する。
        *   **Micro-Costing (マイクロ原価管理):** 定期的に支出の構成要素（ユニット単価：米1杯、シャンプー1回等）まで掘り下げてモデル化し、生活コストの構造的な削減と予算精度の向上を行う。
    3.  **Goal-Based Asset Allocation (ゴールベース運用):** 漠然と資産を増やすのではなく、定義されたライフゴール（住宅、教育、老後等）の必要額から逆算してリスク資産配分を決定する。

*   **Scope Definition (管理境界):**
    *   **Core Scope:** 流動資産、投資資産、実物資産（耐久消費財）、および **「将来負債（Future Liabilities）」**。
    *   **Asset Recognition Threshold (資産計上基準):**
        *   管理コストと効果のバランスを最適化するため、以下の基準を満たすもののみを「資産（Asset）」として計上し、それ以外は購入時に全額を「費用（Expense）」として処理する。
        *   **基準:** 取得価額 **30,000円以上** かつ 想定耐用年数 **1年以上**。
    *   **Concept:**
        *   **Personal ALM (Asset Liability Management):** 従来の「資産管理」を超え、将来のライフイベント（結婚、住宅購入、老後生活費など）を「個人の負債（返済義務のある借金のようなもの）」と見なし、それに対する資産の充足度を管理する手法を採用する。
        *   耐久消費財は費用（Expense）ではなく、資産（Asset）として計上し、日次単位での減価償却コストを管理する。

*   **Metrics (KPI - 成功指標):**
    *   **Roadmap Progression (ロードマップ乖離率):** 長期資産形成ロードマップ（例: 20年後の目標資産額へのパス）に対する現在の資産額の乖離（On Track / Off Track判定）。
    *   **Savings Rate (貯蓄率):** (Free Cash Flow ÷ Income) > 20% (Target)
    *   **ゴール達成確率:** （モンテカルロ・シミュレーション等による、主要ライフイベント資金の準備完了確率） > 85%
    *   **アイドルマネー率:** （役割未定義の現金 ÷ 総現金） = **Target 0%**
    *   **流動性カバレッジ比率:** （流動資産 ÷ 3ヶ月以内の支出予測額） > 100%
    *   **資産寿命:** 現在の資産取り崩しペースでの枯渇予測年月

## 3. 第3層: Functional Requirements (オペレーション / How)
**定義:** 戦略を自動的・効率的に実行するための、具体的なシステム機能要件。

### 3.1. Core Modules
*   **Asset Aggregator (全資産統合):**
    *   銀行、証券、クレジットカード、電子マネー、手元現金の残高を自動取得・統合する。
    *   異なる金融機関のデータを統一フォーマットに正規化する。
*   **B/S Builder (バランスシート生成):**
    *   取得データを元に、リアルタイムで個人の貸借対照表（B/S）を生成する。
    *   資産（流動/固定/投資）と負債（クレカ未払分/ローン/将来のイベント費用）を可視化する。
*   **Cash Flow Analyzer (収支構造分析):**
    *   **Auto-Categorization:** 支出を「基礎生活費（Fixed/Essential）」と「変動費（Variable/Discretionary）」に自動分類する。
    *   **Unit Economics Modeler (生活原価モデラー):** 日々の記録とは別に、詳細な「標準生活原価（米1杯、シャンプー1回等のコスト）」をパラメータとして保持する。予算策定の基礎データとして使用し、定期的に見直す。
    *   **FCF Monitoring:** 毎月の「投資余力（Free Cash Flow）」を算出し、Simulation Engineの入力値（積立能力）を動的に補正する。
    *   **Burn Rate Alert:** 収入が途絶えた場合に、現在の資産で何ヶ月生き延びられるか（Runway）を、生活費の実績値から計算する。
*   **Life Event Planner (ライフイベント＆ゴール管理):**
    *   **Event Definition:** 「5年後に500万円の車」「20年後に2000万円の老後資金」といった将来の支出イベントを登録する。
    *   **Bucket Mapping:** 総資産を「住宅用」「教育用」「余剰資金」などの仮想バケツに紐付け、ゴールごとの進捗率を管理する。
*   **Simulation Engine (確率論的シミュレーション):**
    *   **Probabilistic Forecasting:** インフレ率や運用益の変動（ボラティリティ）を考慮し、モンテカルロ・シミュレーションを用いて「ゴールの達成確率」を算出する。
    *   **Gap Analysis:** 目標額に対する不足分を計算し、「月々の積立額をあと3万円増やすべき」「リスク許容度を上げるべき」といった逆算提案を行う。
*   **Asset Lifecycle Manager (物品管理):**
    *   購入した物品（PC、スマホ、家具等）を「資産」として登録し、減価償却と買い替えアラートを管理する。

### 3.2. User Workflow (To-Be)
1.  **Daily (Monitoring):**
    *   システムがバックグラウンドで全口座情報を同期。
    *   ユーザーは「異常値（予算超過、不正利用の疑い）」の通知があった場合のみ対応。
2.  **Event-Driven (Decision Making):**
    *   **Purchase Impact Check:** 高額商品の購入検討時、商品情報を入力すると、システムが「それを買うと、**5年後の住宅購入ゴールの達成確率が何%下がるか**」というトレードオフを提示する。
3.  **Monthly (Execution):**
    *   システムが「月次レポート（B/S推移、ゴール達成確率の変化）」と「アクションリスト（資金移動指示）」を生成。
    *   ユーザーは内容を確認し、承認することで資産配分（リバランス）を完了させる。
4.  **Quarterly (Optimization):**
    *   **Cost Structure Review:** 「Unit Economics Modeler」を用いて生活標準原価を見直し、予算設定（パラメータ）を更新する。実績とパラメータの乖離が大きい費目を特定して改善する。

## 4. Key Use Cases (主要な利用シナリオ)

### UC-01: Intelligent Monthly Review (月次決算の自動化)
*   **Trigger:** 月初 1日 09:00 (自動通知)
*   **Process:**
    1.  **Auto-Report:** 前月の収支と資産増減、各ライフゴールの進捗状況（達成確率）をレポート化。
    2.  **Anomaly Detection:** 異常支出の検知。
    3.  **Action Proposal:** 余剰資金の投資配分や、ゴール達成率低下時のリカバリー案（節約目標の提示など）を作成。
    4.  **Decision:** ユーザー承認による資金移動実行。

### UC-02: Smart Purchase Audit (AI購買監査・トレードオフ分析)
*   **Concept:** 欲しいモノを買うことによる「未来の犠牲」を可視化する。
*   **Trigger:** ユーザーが **Asset Recognition Threshold (3万円以上)** を超える高額商品のURLをチャットボットに送信した時。
*   **Process:**
    1.  **Analysis:** 商品情報の解析（価格、予想リセールバリュー、日割りコスト）。
    2.  **Trade-off Simulation:**
        *   「この20万円を使うと、資産寿命が1ヶ月縮まります」
        *   「または、3年後の海外旅行予算の達成確率が90%→82%に低下します」
    3.  **Recommendation:** トレードオフを踏まえた上での「Go/Wait/Stop」判定。

### UC-03: Dynamic Rebalancing (動的リバランス)
*   **Trigger:** 市場変動またはライフプラン変更時。
*   **Process:**
    1.  **Alert:** 資産配分の乖離、またはゴールの優先順位変更による再計算が必要な場合。
    2.  **Optimization:** 各ゴール（バケツ）のリスク許容度に合わせた最適な資産配分を再計算。
    3.  **Execution:** 課税を考慮した売買指示の生成と実行。
